Lifecycle Hooks — Step by Step Execution Order

Phase 1 — Component BIRTH 🐣

First time component loads:

STEP 0️⃣ — Constructor
━━━━━━━━━━━━━━━━━━━━━━━━━
→ Class is created
→ DI services injected
→ Nothing else ready yet
→ Runs ONCE only


STEP 1️⃣ — ngOnChanges
━━━━━━━━━━━━━━━━━━━━━━━━━
→ Runs ONLY if @Input() exists
→ First time @Input value arrives from parent
→ Gives you OLD value and NEW value
→ Runs BEFORE ngOnInit
⚠️ Skipped if NO @Input() in component


STEP 2️⃣ — ngOnInit
━━━━━━━━━━━━━━━━━━━━━━━━━
→ Component fully initialized
→ @Input() values are ready
→ API calls go here
→ Data setup goes here
→ Runs ONCE only
→ YOUR MOST USED HOOK ⭐


STEP 3️⃣ — ngDoCheck
━━━━━━━━━━━━━━━━━━━━━━━━━
→ Angular checks for changes
→ Runs after ngOnInit
→ Runs after every change detection
→ Very frequent — use carefully ⚠️

STEP 4️⃣ — ngAfterContentInit
━━━━━━━━━━━━━━━━━━━━━━━━━
→ ng-content (projected content) loaded
→ Like React's children prop ready
→ Runs ONCE only


STEP 5️⃣ — ngAfterContentChecked
━━━━━━━━━━━━━━━━━━━━━━━━━
→ ng-content checked by Angular
→ Runs after every content check
→ Runs right after ngAfterContentInit


STEP 6️⃣ — ngAfterViewInit
━━━━━━━━━━━━━━━━━━━━━━━━━
→ Component's HTML template fully rendered
→ DOM is ready
→ @ViewChild is available here
→ Third party libs init here (charts, maps)
→ Runs ONCE only


STEP 7️⃣ — ngAfterViewChecked
━━━━━━━━━━━━━━━━━━━━━━━━━
→ View checked by Angular
→ Runs after every view check
→ Runs right after ngAfterViewInit

Console Output on First Load

0️⃣ Constructor ← class created
1️⃣ ngOnChanges ← @Input arrived
2️⃣ ngOnInit ← component ready
3️⃣ ngDoCheck ← change check
4️⃣ ngAfterContentInit ← content ready
5️⃣ ngAfterContentChecked ← content checked
6️⃣ ngAfterViewInit ← DOM ready
7️⃣ ngAfterViewChecked ← view checked

==============================================================

Phase 2 — Component LIFE 🔄

When something CHANGES (click, input, data update):

Every time change happens:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3️⃣ ngDoCheck ← checks what changed
5️⃣ ngAfterContentChecked ← checks content again
7️⃣ ngAfterViewChecked ← checks view again

If @Input() changed:
1️⃣ ngOnChanges ← fires first!
then above 3 run again

===========================================================

Phase 3 — Component DEATH 💀

When component removed from DOM:

STEP 8️⃣ — ngOnDestroy
━━━━━━━━━━━━━━━━━━━━━━━━━
→ Component about to be removed
→ Clean up everything here
→ Unsubscribe observables
→ Clear timers
→ Remove event listeners
→ Runs ONCE only

Runs ONCE vs Runs MANY TIMES

Runs ONCE:
━━━━━━━━━━━━━━━━━━━━━

2️⃣ ngOnInit
4️⃣ ngAfterContentInit
6️⃣ ngAfterViewInit
8️⃣ ngOnDestroy

━━━━━━━━━━━━━━━━━━━━━
1️⃣ ngOnChanges (every @Input change)
3️⃣ ngDoCheck (every change detection)
5️⃣ ngAfterContentChecked(every change detection)
7️⃣ ngAfterViewChecked (every change detection)

💡 Simple rule — Birth → Life → Death. Constructor and ngOnInit for birth setup,
change hooks for life,
ngOnDestroy for death cleanup.
You'll use 4 hooks 90% of the time — ngOnInit, ngOnChanges, ngAfterViewInit, ngOnDestroy.
