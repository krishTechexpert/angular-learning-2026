Creating Custom pipe

Same as built-in pipes but you build it for your own needs.

Syntax: value | pipeName:param1:param2:param3

Rules of custom pipe | manual method

1.) define class that implements PipeTransform --> write the logic in transform method

2.) decorate the class with @Pipe()

3.) register the class in declarations[] in app.module.ts

Use the pipe in html file

2 Types of pipes

Pure pipe --- stateless by default and executed when pure primivite value/reference changes

Stateless pipes(pure) are pure function that flow input data through without remembering anything
or causing detectable side effects. Most pipes are stateless.

impure pipe --- statefull by default and execute for every change detection such as search on each key stroke

Statefull pipes (impure) are those which can manage the state of the data they transform.
A pipe that creates an http request,store the response and display the output, is a statefull pipe.
Statefull pipe should be used cautiously.
Angular provide Asyn pipe, which is statefull.

By default all predefined pipes such as date,number,json,lowercase are pure pipes

execpt async pipe is impure pipe

we explain those in further with basic details

Complex Example -02 Filter | using Pipe

--------------------------------------------------

By default all pipes are pure:

Pure pipe only runs when input reference changes:

pure pipe runs when:

✅ new array assigned → userList = [...newUsers]

✅ new string typed → filterBy = "kri"

pure pipe does NOT run when:

❌ item added to same array → userList.push(newUser)

❌ item removed from array → userList.splice(0,1)

❌ item property changed → userList[0].name = 'new name'

================================================

Why? — Reference vs Value

// Same reference — pure pipe IGNORES this
userList.push(newUser)
// userList reference is still SAME array
// Angular says "nothing changed" → pipe does NOT run
// New reference — pure pipe SEES this
userList = [...userList, newUser]
// userList is now NEW array
// Angular says "something changed" → pipe runs ✅

💡 Exactly like React — useState re-renders only when reference changes. arr.push() doesn't trigger re-render, setArr([...arr, item]) does.

💡 Best practice — always use new array reference (spread operator) instead of pure: false. Same rule applies in React too — never mutate state directly! 

💡 Same logic — Angular pipe just keeps it out of the component and in the template. Cleaner component, reusable filter anywhere.

Open browser console — you'll see:

pipe run ← fires when component loads

pipe run ← fires every keystroke in input

pipe run ← fires every keystroke in input

💡 This is why trackBy matters with pipes — pipe re-runs on every keystroke.
For large arrays, add pure: false carefully or debounce the input.

React Same Rule

// ❌ React — same bug
users.push(newUser)
setUsers(users) // same reference → no re-render
// ✅ React — correct
setUsers([...users, newUser]) // new reference → re-render
