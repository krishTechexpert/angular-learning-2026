2 types of directives
a.) structural directives --> modify the DOM structure b.) attribute directives --> modify the UI--> apply some css and modify the UI of the elements

Rules of directives in angular

1.) component have to load the directives as directives can not run on their own
(means that esko compoment k ander load karty hai, akely load nhi hota hai)

2.) 1 tag can load only 1 structure directive

3.) 1 tag can load multiple attribute directive

4.) While creating custom directives

4.a.) define class -> decorated with @Directive()

4.b.) @Directive(selector: '[]') selector that is used to calling the directives

4.c.) Custom directives has to be registered in app.modules.ts inside declarations: section. (in angular 15)

4.d.) you can generate directive by angular CLI command by:
ng generate directive Nameofthedirective
or short form as ng g d Nameofthedirective

ElementRef: it is class — Direct DOM Access Think of it like React's useRef().current

Problem with ElementRef — it directly touches the DOM which is:

❌ Unsafe — XSS attacks possible

\❌ Breaks Server Side Rendering (SSR)

❌ Breaks Web Workers

❌ Angular team says avoid it

=======================================================

Renderer2: it is class — Safe DOM Access

Angular's safe wrapper around DOM manipulation.

Simple Analogy

ElementRef = touching live electric wire directly ⚡ dangerous

Renderer2 = using insulated gloves to touch it 🧤 safe

@HostBinding: bind property/class/style to host element in Directives

💡 Rule of thumb — in directives, always prefer @HostBinding + @HostListener over ElementRef + Renderer2. Cleaner code, no direct DOM touching, Angular manages everything.
